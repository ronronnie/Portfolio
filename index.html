How I Work

Design Process

01

Discover

Every great product starts with deep curiosity. I immerse myself in the problem space — conducting stakeholder interviews, user research, competitive audits, and heuristic analyses to build a clear picture of what actually needs solving.

User Research Stakeholder Interviews Competitive Analysis Heuristic Audit
02

Define

Insights become strategy. I synthesise research into a sharp problem statement, map out information architecture, define success metrics, and align all stakeholders around a shared direction before a single pixel is drawn.

Problem Framing Information Architecture User Flows KPI Definition
03

Design

From rough sketches to high-fidelity prototypes. I explore broadly, iterate fast, and validate early — obsessing over typography, spacing, motion, and the micro-interactions that make interfaces feel alive and intuitive.

Wireframing Prototyping Motion Design Design Systems Usability Testing
04

Develop

Design only matters if it ships. I write clean, performant production code — bridging the gap between design intent and engineering reality. Every animation, interaction, and edge case is accounted for before launch.

React / Next.js GSAP / Three.js Performance Optimisation Accessibility
05

Deliver

Launch is just the beginning. I monitor real-user behaviour, run A/B tests, and iterate continuously — turning data into improvements that compound over time and keep the product ahead of the curve.

A/B Testing Analytics Iteration Handoff & Docs
